# Seoul-Bike-Demand-Prediction
Seoul Bike Demand Prediction

## 1. Project Title

Seoul Bike Demand Prediction using Machine Learning and Streamlit

---

## 2. Project Overview

This project is developed to predict the number of rented bikes in Seoul based on weather, time, season, and holiday-related information.

Bike sharing systems are commonly used in smart cities. The demand for bikes changes based on temperature, humidity, rainfall, snowfall, season, time of day, and holidays. By predicting bike demand in advance, bike rental companies can manage bike availability more efficiently.

In this project, I used machine learning regression models to predict the rented bike count. I also created a simple Streamlit web application where users can enter input values and get the predicted bike demand.

---

## 3. Objective of the Project

The main objective of this project is to build a machine learning model that can predict bike rental demand.

The specific objectives are:

- To understand the Seoul Bike Sharing Demand dataset.
- To clean and prepare the data for machine learning.
- To perform feature engineering using date and time columns.
- To analyze bike demand using graphs.
- To train regression models for prediction.
- To compare model performance using evaluation metrics.
- To save the best model.
- To create a Streamlit web app for user-friendly prediction.

---

## 4. Problem Statement

Bike rental demand is not constant throughout the day. It depends on many factors such as weather, season, holiday, and time.

The problem is to predict the number of rented bikes for a given set of input conditions.

This is a regression problem because the target value is numerical.
